    使用说明

    这份文档把你从「拿到这个仓库」一路带到「能写、能发布、能被评论」。按顺序读一遍就会用了。

    0. 你拿到了什么

    一个开箱即用的 MkDocs（Material 主题）笔记站，已经配好了你要的全部功能：

    • 中文界面、深/浅色切换
    • 折叠块（解释专有名词）、提示框
    • 代码高亮 + 一键复制（代码不需运行，写进 md 即可）
    • 数学公式、流程图（mermaid）
    • 复选框（尝试打勾）、标签检索、脚注
    • 按日期的博客（科研日记 / 开发日志 / 每日阅读）
    • 评论（giscus，必须登录 GitHub 才能评论）
    • 每页自动显示更新时间
    • push 到 GitHub 即自动构建发布

    仓库里已经放了示例页，照着改就行，不用从零写。

    1. 准备工作（只做一次）

    你需要本机有 Python 3（3.9 以上）。检查：

    python3 --version

    进入仓库目录，安装依赖：

    cd my-site
pip install -r requirements.txt

    Windows 用户把 python3 换成 python。如果 pip 报权限问题，加 --user。

    2. 本地预览（你每天用得最多的命令）

    mkdocs serve

    然后浏览器打开 http://127.0.0.1:8000 。这个命令会热重载——你改任何 .md 文件保存后，网页自动刷新。写笔记时就一直开着它。

    按 Ctrl + C 停止。

    3. 目录结构（每个东西是干嘛的）

    my-site/
├─ mkdocs.yml              ← 站点总配置（站名、导航、主题、插件都在这）
├─ requirements.txt        ← 依赖清单
├─ 使用说明.md             ← 就是本文件
├─ .github/workflows/
│   └─ deploy.yml          ← 自动部署脚本（push 即触发，一般不用动）
├─ overrides/partials/
│   └─ comments.html       ← giscus 评论配置（部署评论时改这里）
└─ docs/                   ← 你所有的笔记都写在这个文件夹里
    ├─ index.md            ← 首页（理念 + TODO）
    ├─ courses/            ← 课程笔记（NJU OS / CS336 / 编译）
    ├─ labs/               ← 实验/踩坑记录（预计最大块）
    ├─ research/           ← 科研：LLM Wiki 进度 + 论文解读
    ├─ blog/posts/         ← 博客帖子（日记/日志/阅读）
    ├─ help/               ← 功能速查表 + 踩坑模板（写笔记忘了语法就来这）
    ├─ assets/pdfs/        ← 放 lecture-to-notes 生成的 PDF
    ├─ javascripts/        ← MathJax 配置（不用动）
    └─ stylesheets/        ← 自定义样式（想微调外观再动）

    核心心法：你 99% 的时间只在 docs/ 里写 .md 文件，偶尔动一下 mkdocs.yml 的导航。

    4. 怎么写一篇新笔记

    分三步：

    第 1 步：新建一个 .md 文件

    比如给 CS336 写第二讲，就在 docs/courses/cs336/ 下建一个 lecture-02.md，开头写个标题：

    # 第 2 讲：xxx

正文……

    第 2 步：把它加进导航

    打开 mkdocs.yml，找到 nav:，在对应位置加一行（注意缩进用空格）：

      - 课程笔记:
      - courses/index.md
      - Stanford CS336: courses/cs336/index.md
      - CS336 第 2 讲: courses/cs336/lecture-02.md    # ← 新增这行

    存盘后 mkdocs serve 的页面会自动出现这一项。

    第 3 步：用上各种功能

    所有语法的写法 + 效果都在 docs/help/feature-snippets.md（站点里的「功能速查表」）。最常用的几个：

    • 折叠解释：??? note "展开：术语" 然后缩进写内容
    • 代码：用三个反引号包起来，标上语言
    • 公式：行内 \( ... \)，整行 \[ ... \]
    • 复选框：- [ ] 和 - [x]

    忘了就去翻速查表，不用背。

    5. 几类特定笔记怎么写

    写一条踩坑记录（实验区）

    1. 在 docs/labs/<课程>/ 下新建 .md。
    2. 复制 docs/help/lab-entry-template.md 里的模板填空（现象→环境→弯路→根因→解法→教训）。
    3. 顶部 frontmatter 写好 tags（决定它进哪些标签分类）和 comments: true（要不要评论）。
    4. 加进 mkdocs.yml 的导航。

    参考已有的示例：docs/labs/nju-os/lab-01-example.md。

    记住两层制：调试时的流水草稿可以只留本地；只有值得复用的坑才提炼成正式条目放进来。

    写一篇日志（科研日记/开发日志/阅读）

    在 docs/blog/posts/ 下新建文件，顶部必须有 date：

    ---
date: 2026-06-08
categories:
  - 每日阅读
---

    6. 部署上线 ⭐（重点，请仔细读）

    一个必须先知道的前提

    GitHub 官方规则：免费账号只能给「公开仓库」开 Pages。要让「私有仓库」发布公开站点，需要 GitHub Pro。

    你是学生 → 去申请 GitHub 学生开发包（Student Developer Pack），它免费给你 Pro，正好解锁私有仓库的 Pages。这是为你这套需求量身定做的免费路径。

    路线一（推荐）：学生包 + GitHub Pages

    步骤 1：拿 Pro。 访问 https://education.github.com/pack，用学校邮箱/学生证认证，通过后账号自动升级到 Pro。

    步骤 2：建私有仓库并推上去。 在 GitHub 网页新建一个 Private 仓库（先不要勾任何初始化文件），然后在本地：

    cd my-site
# 部署前先把 mkdocs.yml 里的 <你的名字> 等占位符改好
git init
git add -A
git commit -m "init my notes site"
git branch -M main
git remote add origin git@github.com:<你的用户名>/<仓库名>.git
git push -u origin main

    步骤 3：等自动构建。 push 后去仓库的 Actions 标签，会看到 deploy 工作流在跑。它跑完会自动创建一个 gh-pages 分支（里面是构建好的站点）。

    如果 Action 报权限错误：去 仓库 Settings → Actions → General → Workflow permissions，选 Read and write permissions，保存后重跑。

    步骤 4：开启 Pages。 去 仓库 Settings → Pages → Source 选 Deploy from a branch → Branch 选 gh-pages / (root) → Save。

    步骤 5：访问。 等一两分钟，站点就在 https://<你的用户名>.github.io/<仓库名>/。

    步骤 6（建议）： 回到 mkdocs.yml，把 site_url 那行的注释去掉并填成上面的地址，再 push 一次。这样搜索、sitemap 等会更准。

    之后你每次 push 到 main，站点都会自动更新，不用再手动做任何事。

    路线二（不想用 Pro）：Cloudflare Pages

    Cloudflare Pages 免费、且支持从私有仓库构建：

    1. 登录 Cloudflare → Workers & Pages → 创建 → Pages → 连接你的 GitHub 私有仓库。
    2. 构建命令填：pip install -r requirements.txt && mkdocs build
    3. 输出目录填：site
    4. 加一个环境变量 PYTHON_VERSION = 3.12。
    5. 部署完成后给你一个 *.pages.dev 地址。

    走这条路就不需要 .github/workflows/deploy.yml 了（可留着不管）。

    7. 配置评论（giscus）

    评论的后端是 GitHub Discussions，需要一个公开仓库来存评论（你的笔记主仓库保持私有不受影响）。

    1. 建一个公开仓库专门放评论（比如叫 site-comments），在它的 Settings → General 勾选启用 Discussions。
    2. 给这个仓库安装 giscus App：访问 https://github.com/apps/giscus → Install → 选这个仓库。
    3. 去 https://giscus.app ，在页面里填入这个公开仓库，映射方式选 pathname，分类随便选一个（比如 Announcements）。页面下方会生成一段 <script>，里面有四个值：data-repo、data-repo-id、data-category、data-category-id。
    4. 打开 overrides/partials/comments.html，把里面四个 <...> 占位符替换成这四个值。
    5. 想开评论的页面，在顶部 frontmatter 加 comments: true（示例 lab 已经加了）。不加的页面就没有评论。

    配好后，访客点评论时会被要求用 GitHub 登录——这正好挡掉路人乱评论。

    8. 个性化

    都在 mkdocs.yml 里：

    • 站名：改 site_name。
    • 作者：改 site_author 里的 <你的名字>。
    • 主题色：把两处 primary: teal / accent: teal 换成别的色（如 indigo、deep purple、pink）。
    • 顶部分区顺序：调整 nav: 的顺序。

    想微调字体/间距等细节，改 docs/stylesheets/extra.css。

    9. 常见问题

    • mkdocs: command not found：依赖没装好，重跑第 1 步的 pip install。
    • 改了文件但页面没变：确认 mkdocs serve 还开着；硬刷新浏览器（Ctrl+F5）。
    • 新建的页面没出现在导航：检查有没有加进 mkdocs.yml 的 nav:，注意缩进是空格不是 Tab。
    • 构建时一堆红色 MkDocs 2.0 警告：那是 Material 团队的固定提示，不影响使用，忽略即可。
    • 公式不显示：检查是否用了 \( \) / \[ \]（不是美元符号）。
    • 评论不显示：确认页面有 comments: true，且 comments.html 里四个值都填了、giscus App 已装到公开仓库。
    • 千万别把 site/ 文件夹提交进 git（已在 .gitignore 里忽略，别去动它）。

    10. 把 AI 笔记接进来（可选，之后再弄）

    参考的工具是 lecture-to-notes（视频→PDF 讲义）和 paper-to-html（论文→HTML 解读），它们是跑在 Claude Code / Codex 里的 skill，在你本地生成静态文件，再放进 docs/assets/ 链接进笔记。安装方式见其仓库：https://github.com/ysyecust/lecture-to-notes。

    需要时把这步交给我，我带你装好并跑通第一份。

    写完一篇笔记的标准流程就是：建 md → （需要的话）加进 nav → mkdocs serve 看效果 → git add/commit/push → 自动上线。熟练之后整个过程几十秒。
