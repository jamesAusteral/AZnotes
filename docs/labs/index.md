# 实验记录

这是整个站点预计最大的部分。核心原则：**按"问题"组织，不按时间流水账。**

## 两层制写法

为了不让"写笔记"本身变成负担：

1. **原始草稿层**：调试时随便记，时间戳 + 报错 + 试了啥，怎么乱都行。这层甚至可以只留在本地，不上站。
2. **提炼条目层**：事后把真正有价值的坑整理成结构化条目，这层才进站、才打标签、才可被搜索。

不要每个细碎问题都写成精品。**只有"以后还会再踩、或值得别人看"的坑才提炼。**

## 提炼条目长什么样

每条按固定结构写：现象 → 环境 → 走过的弯路 → 根因 → 解法 → 一句话教训。完整模板见 [踩坑条目模板](../help/lab-entry-template.md)，下面是一个真实例子：

## 怎么检索

每条顶部用标签（`tags`）打分类，比如 `xv6`、`并发`、`gdb`。所有标签会自动聚合到[标签索引](../tags.md)，按标签就能找到同类的坑。
