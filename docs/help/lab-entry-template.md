# 踩坑条目模板

写新的实验记录时，复制下面这段，改成你的内容。把这块代码原样拷到一个新的 `.md` 文件里：

````markdown
---
tags:
  - 标签1
  - 标签2
comments: true
---

# 一句话标题（说清是什么坑）

## 现象
出了什么问题，报错长什么样。

## 环境 / 触发条件
版本、配置、什么情况下才触发、是否必现。

## 走过的弯路
试过但没用的方向（这部分很值钱，省得下次重走）。

## 根因
真正的原因是什么。

## 解法
怎么修的，贴关键代码：

```语言
关键代码
```

## 一句话教训
**下次遇到类似情况，记住这一句。**
````

要点：

- `tags` 决定它出现在[标签索引](../tags.md)的哪些分类下。
- `comments: true` 给这一页开启评论（不写就没有评论）。
- 标题用一句话讲清是什么坑，方便以后搜。
